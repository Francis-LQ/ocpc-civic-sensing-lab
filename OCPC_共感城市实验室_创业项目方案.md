
# 「共感城市实验室」OCPC 素养教育创业项目方案

> 任务目标：基于开源、艺术、博物馆、教育、硬件电路、创客教育、媒介艺术与公民科学相关文献，设计一个面向青少年与大学生的「艺术教育 × 科学传播 × 开放公民项目公地」素养教育创业项目。  
> 核心判断：前台品牌采用「共感城市实验室 / Civic Sensing Lab」，后台方法论与项目孵化框架采用 **OCPC / Open Civic Project Commons / 开放公民项目公地**。

---

## 0. 执行摘要

**推荐项目名称**：共感城市实验室  
**英文名称**：Civic Sensing Lab  
**缩写**：CSL  
**底层框架**：Open Civic Project Commons，简称 OCPC  
**一句话定位**：面向青少年与大学生，用开源硬件、环境数据、媒介艺术和公共展览，把真实世界的公共议题转化为可学习、可展示、可复用、可持续维护的开放公民项目。

该项目不是普通 STEAM 课程，也不是一次性创客工作坊。它的核心不是“教会学生做一个小作品”，而是把学生、教师、馆员、研究者、社区、企业 CSR/ESG、公益组织和地方公共机构连接成一个长期运行的 **开放公民项目公地**：每个项目都要留下问题陈述、数据采集方案、硬件 BOM、电路图、代码、艺术表达方案、展示文档、伦理说明、开放许可、贡献记录和复用指南。

**商业前台**负责交付课程、训练营、策展、研学、黑客松、教师馆员培训和 B2B/B2G/B2M 项目；**OCPC 后台**负责项目资产沉淀、开源文档、贡献治理、公共价值记录、项目复用与社区维护；**共荣文化**负责保护学生、导师、志愿者和贡献者的成长节奏、尊严、能量与退出权。

## 2. 趋势与机会洞察

| 趋势 | 文献依据 | 机会判断 | 对教育产品的启发 | 对 OCPC 的启发 |
|---|---|---|---|---|
| 1. 从 STEAM 走向“艺术 × 科学 × 社会议题 × 公共参与” | 电路课程强调工科与艺术交叉、非技术因素；开源策展强调社会行动网络；共享策展关注公众参与技术事务。 | 传统 STEAM 容易停在技能拼盘，公共议题能提高学习动机和社会价值。 | 所有课程都以真实问题启动，如空气、声音、水体、热岛、生物多样性。 | 项目档案必须保留公共议题背景和利益相关方地图。 |
| 2. 从知识传授转向项目制、行动学习和证据链 | 开源文化强调问题解决即知识获得；Mind+ 研究强调项目流程与学习支架。 | 学生、学校和机构都需要能展示学习过程的证据。 | 课程设计为“观察—问题—数据—原型—展示—复盘”。 | OCPC 保存过程证据，而不是只保存最终作品。 |
| 3. 从单次工作坊转向长期社区与项目公地 | 任务文档明确要求长期沉淀项目、知识、工具、数据、社区和公共价值。 | 一次性活动难以形成品牌与复购，项目库能形成壁垒。 | 工作坊是入口，训练营和孵化是连续路径。 | 项目必须有维护人、版本号、许可和复用指南。 |
| 4. 从博物馆参观转向策展实验室和公共教育共创 | 非遗研究提出“策展实验室”和社区化融合；开源策展提出开源协同与知识共享。 | 博物馆教育部门需要可持续、可传播、可共创的公共教育项目。 | 与馆方联合开发数据装置、社区调查、展教合一课程。 | 博物馆成为 OCPC 线下节点和公共展示界面。 |
| 5. 从创客工具教学转向真实世界复杂问题解决 | 电路课程以复杂工程问题为核心，并纳入工程伦理、环境保护等非技术因素。 | 工具课同质化严重，真实议题项目更有差异化。 | 不卖“Arduino 入门”，而卖“空气数据艺术项目”“校园热岛行动”。 | 项目模板中加入安全、伦理、隐私、社会影响评价。 |
| 6. 从艺术表达转向数据、环境、媒介和硬件的复合表达 | 媒体艺术文献把数据、代码和科学可视化纳入艺术；物件剧场强调空间叙事。 | 青少年对可视化、声音、互动装置、短视频接受度高。 | 让数据成为灯光、声音、地图、影像、展签和互动装置。 | OCPC 需支持数据、代码、图片、音频、视频、模型和展陈文档。 |
| 7. 从学生作品展示转向开放项目档案与贡献记录 | 开源策展强调开放内容、知识共享和档案实践；任务文档要求贡献记录。 | 学生需要升学/简历可证明的真实项目经历。 | 每个项目结课时生成项目页和贡献页。 | 建立贡献记录、项目版本、引用规范和复用统计。 |
| 8. 从竞赛式创新转向孵化、维护和公共价值分配 | 黑客松常见问题是赛后消失；任务文档要求 OCPC 孵化。 | 机构客户更关注持续影响，而非一次热闹。 | 赛后设置 30/60/90 天维护任务和微型资助。 | 孵化阶段从 Issue Discovery 到 Commons Release 再到 Incubation。 |
| 9. 从个人英雄创业转向法律外壳 + 公地治理 + 共荣文化 | 任务文档强调法律主体与 OCPC 分离；公司法要求公司承担社会责任。 | 公共教育项目必须兼顾商业效率与公共信任。 | 早期用公司交付，公共成果进入 OCPC。 | 设顾问委员会、伦理审查、贡献者权利和退出机制。 |

### 4.1 项目名称与定位

- **英文名称**：Civic Sensing Lab

- **一句话介绍**：共感城市实验室通过开源硬件、环境数据、媒介艺术和公共展览，帮助青少年与大学生把身边公共问题变成可验证、可表达、可复用、可持续维护的开放公民项目。

**100 字以内简介**：  
共感城市实验室是一套面向青少年与大学生的艺术科技素养教育项目，围绕空气、声音、水体、热岛、生物多样性和公共空间等议题，结合开源硬件、数据采集、媒介表达、公共展览和 OCPC 项目公地孵化，培养学生科学观察、证据意识、艺术表达、开放协作和公共参与能力。

**300 字以内愿景**：  
我们希望把学校、博物馆、科技馆、社区、公园、科研机构、开源社区和公益资源连接起来，让青年不再只是被动学习科学知识，也不只是完成一次创客作品，而是能够用证据理解真实世界，用艺术表达公共经验，用开源方式分享工具与知识，用共同治理维护项目成果。长期看，共感城市实验室将沉淀一批可复用的青少年公民科学项目、环境数据工具包、开放课程、数据艺术展览、公共议题档案和贡献记录，形成面向公共利益的青年 OCPC 项目网络。

### 6.3 黑客松模型

- **名称**：共感城市黑客松 / Civic Sensing Hackathon
- **主题**：用数据、硬件与艺术回应一个身边公共议题。
- **参赛对象**：高中生组、大学生组、混合公众组。
- **赛题来源**：学校、博物馆、社区、公园、公益组织、科研顾问提出；所有赛题必须经过安全与伦理预审。
- **数据来源**：现场采集数据、开放公共数据、机构授权数据、学生自建传感器数据；涉及个人信息的数据默认不公开。
- **硬件工具包**：ESP32/Arduino、温湿度、PM、CO2、光照、噪声、水质基础传感器、GPS/地图模板、移动电源、安全线材。
- **艺术表达要求**：最终必须有一个面向公众的表达界面，如灯光装置、数据海报、声音地图、互动页面、影像短片或物件剧场。
- **公共利益技术要求**：必须说明受影响群体、公共价值、潜在风险、数据最小化原则和维护计划。
- **导师构成**：科学导师、硬件导师、数据导师、艺术/策展导师、公共议题导师、伦理与安全导师。
- **评审标准**：科学问题清晰度 15%；数据与证据质量 15%；艺术表达质量 15%；公共议题相关性 15%；原型可行性 10%；开源文档完整度 10%；社区贡献潜力 10%；伦理、安全与隐私合规 5%；共荣协作表现 5%。
- **奖项设置**：最佳公共问题奖、最佳数据证据奖、最佳艺术表达奖、最佳开源文档奖、最佳社区潜力奖、共荣协作奖。
- **后续孵化**：获奖项目进入 90 天 OCPC Mini-Incubation，提供导师时段、设备支持、微型展示机会和版本维护任务。
- **防止项目消失**：结营前必须完成 Commons Release 最小包：README、BOM、代码/伪代码、数据样本、展签、风险说明、贡献者名单、下一步维护人。

### 6.4 开源社区设计

- **社区名称**：CSL Open Commons Community / 共感开放公地社区
- **社区使命**：把青年公共议题项目从“一次作品”变成“可复用公共知识”。
- **开源内容类型**：课程包、硬件设计、代码、数据采集协议、艺术表达模板、展览脚本、评价量表、风险清单。
- **线上平台**：Gitee/GitHub 项目库 + Notion/语雀知识库 + 飞书/微信群协作 + 官网项目页。未成年人参与的实名信息不公开。
- **线下活动**：月度项目夜、季度开放采样日、年度 OCPC 展、文档马拉松。
- **角色体系**：学习者、贡献者、项目维护者、导师、数据管理员、伦理安全员、策展人、社区协调员、顾问委员。
- **贡献机制**：Issue、Pull Request、文档提交、数据提交、复现实验、翻译、本地化、展览志愿服务。
- **激励机制**：贡献徽章、公开致谢、项目引用、推荐信、展示机会、导师办公时间、微型资助。
- **开放许可策略**：文本/图片优先 CC BY-NC-SA 或 CC BY；代码 MIT/Apache-2.0；硬件 CERN-OHL 或 TAPR OHL；数据优先发布去标识化聚合数据，必要时不开放原始数据。
- **保护机制**：未成年人使用别名公开；贡献需要监护人授权；禁止公开个人轨迹、面部、联系方式；设置“暂停贡献”和“撤回署名/匿名化”流程。

**项目库结构模板**：项目简介、公共议题背景、科学问题、利益相关方地图、数据采集方法、硬件 BOM、电路图、代码仓库、艺术表达方案、展示文档、教学包、安全与伦理说明、开放许可、贡献者名单、复用指南、版本日志、维护人。

### 6.6 OCPC 项目公地孵化机制

**OCPC 项目定义**：以公共利益议题为核心，具有科学问题、数据或证据、开放原型、公共表达、开放文档、贡献记录和维护计划的青年公民项目。

| 阶段 | 名称 | 核心任务 | 产出物 | 进入下一阶段标准 |
|---|---|---|---|---|
| 0 | Issue Discovery | 发现公共议题 | 问题陈述、利益相关方地图 | 问题真实且有学习价值 |
| 1 | Civic Research | 公民科学调研 | 数据方案、访谈、观察记录 | 有基本证据链 |
| 2 | Open Prototype | 开源原型 | 硬件、代码、媒介原型 | 可被测试和复现 |
| 3 | Public Exhibition | 公共展示 | 展览、工作坊、传播内容 | 能被公众理解和参与 |
| 4 | Commons Release | 公地发布 | 项目档案、许可、贡献记录 | 可复用、可贡献 |
| 5 | Incubation | 孵化与扩散 | 合作、资金、课程化或产品化 | 具备持续维护机制 |

**支持体系**：导师机制、技术支持、艺术策展支持、科学顾问、公共议题顾问、开源发布规范、数据与隐私规范、开放许可策略、贡献记录、展示路演、复用机制、社区维护、微型资助。

## 10. 品牌与传播策略

- **品牌关键词**：真实世界、共感、开源、证据、公共表达、项目公地、青年行动。
- **品牌人格**：好奇、可靠、温暖、有公共责任感、不过度说教。
- **视觉风格**：城市地图线条 + 传感器节点 + 数据渐变 + 手绘自然观察；避免过度科技蓝，增加自然材料和手作质感。
- **中文口号**：一起用艺术和科学观察真实世界。
- **英文口号**：Sense the city. Shape the commons.

| 对象 | 话术 |
|---|---|
| 学生 | 你可以用传感器、地图、声音和影像，把身边的问题做成真正能被看见的项目。 |
| 家长 | 这不是普通手工课，而是科学观察、数据证据、艺术表达和公共协作的综合素养训练。 |
| 学校 | 项目制科学与艺术融合素养课程，可形成校本课程、综合实践和成果展示。 |
| 博物馆 | 让公共教育从展厅延伸为开放项目公地，形成可持续的观众参与和项目档案。 |
| 公益/政府 | 面向公共利益的青年公民科学行动，具备开放成果、可评估指标和传播价值。 |
| 高校 | 跨学科通识教育、创新创业与公共利益技术实践平台。 |
| 企业 CSR/ESG | 支持青年用科学与艺术参与环境和社区议题，形成可公开的公共价值成果。 |

**内容栏目**：一周一个城市问题、传感器小实验、数据变成艺术、学生项目档案、馆校共创案例、OCPC 贡献者故事、安全与伦理小课堂。  
**年度旗舰活动**：Civic Sensing Commons Week / 共感城市项目周，包括开放采样日、数据艺术展、黑客松、项目发布会、教师馆员论坛。  
**OCPC 对外叙事**：这是一个青年公共项目的“共享实验田”，任何人都可以学习、复用、贡献和维护，但必须遵守安全、伦理、开放许可和共荣文化规则。

## 12. 治理、法律与组织结构

### 12.1 三层结构

| 结构 | 功能 | 适合承担的事务 | 不适合承担的事务 | 风险 |
|---|---|---|---|---|
| 法律外壳：一人公司/普通有限公司/非营利组织/合作社 | 合同、财务、责任、雇佣、保险 | 收款、开票、采购、雇佣、商业交付、保险 | 开放项目贡献治理、公共价值代表 | 公司化削弱公共信任；财产混同 |
| OCPC 项目公地 | 开放参与、项目档案、贡献治理、知识沉淀、开放许可 | 项目库、贡献记录、开放文档、公共成果 | 直接收款、雇佣、承担法律责任 | 治理空心化、贡献被滥用 |
| 社区治理机制 | 角色分工、维护、冲突处理、退出和照护 | 贡献者权益、导师边界、项目交接 | 替代法律合规、替代合同 | 权力不平等、志愿者倦怠 |

### 12.2 角色体系

创始人/项目负责人、运营经理、课程负责人、硬件负责人、数据管理员、策展负责人、安全与伦理负责人、社区协调员、项目维护者、学生贡献者、导师、顾问委员、外部审查人。

### 12.3 决策机制

- 商业交付：法律主体负责人决策，重大项目听取顾问意见。
- OCPC 项目发布：维护者 + 数据管理员 + 伦理安全员三方确认。
- 许可变更：需要原始贡献者或其监护人授权边界内进行。
- 未成年人数据：默认不公开个人层面的原始数据。

### 12.4 收益与声誉分配原则

- 商业收入优先覆盖课程研发、导师报酬、保险、安全与平台维护。
- 每个商业项目提取一定比例或固定额度反哺 OCPC 公共版维护；具体比例需结合成本进一步验证。
- 学生、导师、志愿者贡献必须署名或按意愿匿名；商业使用需另行授权。

### 12.5 开放许可策略

| 内容 | 默认策略 |
|---|---|
| 教案/文档 | 公共版 CC BY-NC-SA；商业增强版保留服务权 |
| 代码 | MIT 或 Apache-2.0 |
| 硬件设计 | CERN-OHL-S 或 CERN-OHL-W；简单教程可 CC BY |

## 13. One Person Company / 一人公司维度与中国合规分析

### 13.1 不要混同：一人公司 ≠ OCPC

- **一人公司/单一股东有限责任公司**：法律外壳，负责合同、收款、开票、采购、雇佣、保险和责任边界。
- **OCPC**：开放公民项目公地，负责开放项目、公共档案、贡献治理、知识复用、开放许可和公共价值沉淀。
- **结论**：早期可以用一人公司启动商业交付，但不能把一人公司包装成公共公地；公共信任需要通过顾问委员会、开放规则、财务/贡献透明和公共版成果来补足。

### 13.2 一人公司可行性

| 维度 | 可行性 | 优势 | 风险 | 对策 |
|---|---|---|---|---|
| 注册与启动 | 较高 | 决策快、成本低、适合 MVP | 公共项目显得个人化 | 建立顾问委员会与 OCPC 公开规则 |
| 合同与收款 | 高 | 可签约、开票、采购 | 客户对履约能力有疑虑 | 使用标准合同、保险、项目管理制度 |
| 责任边界 | 中 | 有法人财产边界 | 单一股东财产独立证明压力大 | 单独账户、规范会计、禁用个人账户收款 |
| 融资与团队 | 低到中 | 早期灵活 | 不适合长期股权合作 | 有合伙人/投资时升级普通有限公司 |
| 公益信任 | 中低 | 可执行公益项目 | 易被质疑商业化 | 公益项目通过基金会专项/非营利合作承接 |

### 13.3 组织形式比较

| 形式 | 主要功能 | 优势 | 风险/限制 | 适合本项目阶段 |
|---|---|---|---|---|
| 一人公司/单一股东有限责任公司 | 早期商业交付 | 快速、低成本、责任相对清晰 | 财产混同风险、公共信任不足 | 0–12 个月 MVP |
| 普通有限责任公司 | 团队化经营 | 可引入合伙人、员工、投资 | 治理复杂度提升 | 有稳定客户和合伙人后 |
| 非营利组织 | 公益项目与公共信任 | 适合普惠、公益、社区项目 | 登记门槛、运营合规高 | 有公益资助和长期公共项目后 |
| 基金会专项项目 | 承接公益资金 | 信任强、财务规范 | 受基金会规则约束 | CSR/ESG、普惠科学教育 |
| 合作社 | 成员共同拥有 | 强调共益与协作 | 不一定适合教育科技交付 | 社区节点成熟后可探索 |
| OCPC 项目公地 | 项目资产与开放治理 | 沉淀公共价值、贡献可见 | 不能替代法律主体 | 全阶段都需要 |

### 13.4 推荐组织组合

**0–12 个月**：一人公司 + OCPC 项目库 + 轻量顾问委员会。  
**12–24 个月**：普通有限责任公司 + OCPC 项目公地 + 开源社区。  
**公共项目成熟后**：非营利组织/基金会专项项目 + 服务公司 + OCPC 项目公地。

### 13.5 事务分工

| 事务 | 承担主体 |
|---|---|
| 合同、收款、开票、采购、雇佣、保险 | 法律外壳 |
| 课程交付、训练营安全、供应商管理 | 法律外壳 |
| 开源文档、学生贡献、项目档案 | OCPC + 项目维护者 |
| 公民科学数据治理 | 数据管理员 + 法律主体 + OCPC 规则 |
| 公共展示 | 法律主体承办，OCPC 归档 |
| 商业收入反哺 | 法律主体预算化支持 OCPC 维护 |
| 青少年保护 | 法律主体承担合规，OCPC 规则补充 |

### 13.6 中国合规议题清单

| 类别 | 清单 |
|---|---|
| 公司登记 | 名称、住所、章程、法定代表人、经营范围、股东出资期限、银行账户、会计制度。 |
| 经营范围 | 可考虑教育咨询、科普宣传服务、会议及展览服务、组织文化艺术交流、软件开发、技术服务、文具/电子产品销售；具体表述需当地市场监管核准。 |
| 注册资本 | 避免虚高；按真实履约能力设置；注意有限责任公司认缴出资期限。 |
| 税务 | 发票品类、增值税、企业所得税、个税代扣、成本票、公益捐赠票据边界。 |
| 教育培训监管 | 面向中小学生收费课程可能触发校外培训监管；避免学科培训；按地方非学科培训政策办理资质/备案，需进一步核查地方规则。 |
| 研学安全 | 户外活动保险、交通、带队比例、应急预案、风险告知、监护人同意、医疗应急。 |
| 硬件安全 | 低压用电、焊接/切割限制、设备质检、材料阻燃、实验室规则、眼手防护。 |
| 数据隐私 | PIPL 同意、最小必要、未成年人监护人同意、敏感个人信息评估、数据删除。 |
| 知识产权 | 学生作品授权、导师课件归属、商标、软件著作权、开源许可兼容。 |
| 开源许可 | 代码、硬件、文档、数据分别选择许可；商业版与开源版边界清晰。 |
| 未成年人保护 | 不安排商业性推销；活动适龄；防欺凌、防性骚扰、安全预案。 |
| 公益商业边界 | 企业赞助不等于公益募捐；无公开募捐资格不得自行公开募捐；公益项目可与基金会合作。 |

### 13.7 最终判断

- **早期适合用一人公司启动**：适合验证课程、训练营、工具包和小型策展服务；前提是严格财务隔离、合规经营、购买保险、建立安全制度。
- **何时升级普通有限责任公司**：出现联合创始人、长期员工、外部投资、多个机构客户、跨城市复制或较大额合同后。
- **何时引入非营利组织/基金会专项项目**：有普惠科学教育、公益资助、弱势群体服务、公共数据长期项目或政府/基金会要求公益主体时。
- **公司内部保留**：商业课程增强版、工具包供应链、客户合同、员工和供应商管理、商业营销。
- **开放到 OCPC**：基础课程模板、项目问题库、采样协议、开源硬件设计、去标识化数据样本、展陈说明、贡献记录、复用指南。
- **商业版 + 开源公共版双轨**：公共版保证学习与复用；商业版提供完整交付、导师服务、安全保障、质量控制、材料包和定制化。

## 14. 风险与伦理

| 风险 | 应对策略 |
|---|---|
| 教育效果难评估 | 建立前后测、作品评价、过程证据、同伴互评和长期跟踪。 |
| 户外研学安全 | 风险评估、保险、带队比例、路线预查、天气预案、急救包。 |
| 硬件电路风险 | 低压设备、禁止高危焊接、教师操作高风险工具、安全培训。 |
| 数据隐私 | 数据最小化、去标识化、监护人同意、敏感数据不公开。 |
| 公民科学数据质量 | 标准采样协议、重复测量、异常标注、专家抽检。 |
| 商业化与公益性冲突 | 公开商业/公共版边界，企业赞助不干预结论。 |
| 开源知识产权 | 统一许可模板，贡献者授权确认，商业使用另行协议。 |
| 与学校体系衔接困难 | 提供课标映射、评价表、教案和安全材料。 |
| 孵化后续不足 | 设置 90 天维护期、导师办公时间、微型资助。 |
| 青少年贡献被不当使用 | 未成年人贡献默认非商业授权；商业使用需二次授权。 |
| 公共议题传播失控 | 只做证据充分、适龄、低敏感议题；传播前审查。 |
| 社区权力不平等 | 角色透明、贡献公开、冲突处理、反骚扰规则。 |
| 参与者倦怠 | 任务上限、休息机制、退出权、交接模板。 |

**共荣文化规则**：
1. 成长优先于产出；不以竞赛结果压倒学习过程。  
2. 贡献必须被承认；允许匿名或撤回公开署名。  
3. 学生有退出权；退出不被污名化。  
4. 导师和志愿者有边界；禁止无偿无限加班。  
5. 公共价值不能以牺牲个人隐私、尊严和安全为代价。  
6. 项目交接是权利也是责任；无人维护项目应标记为 archived。
